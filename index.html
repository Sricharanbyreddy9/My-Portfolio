RAGCare – Retrieval-Augmented Healthcare Assistant

An intelligent healthcare chatbot designed to provide accurate, trustworthy answers to medical questions by grounding Large Language Model (LLM) responses in real medical documents. This project showcases the practical application of advanced AI techniques like Retrieval-Augmented Generation (RAG) within a full-stack web application.

Core Functionality & Implementation

  • RAG Architecture: Implements RAG using SentenceTransformers for text embedding and FAISS for efficient vector similarity search. User queries are embedded and matched against chunks of uploaded/preloaded PDF documents to retrieve the most relevant context.
  • LLM Integration: Seamlessly integrates with multiple LLM APIs (OpenAI GPT-4 via openai library, Groq LLaMA 3 via requests) to generate answers based on the retrieved context, offering flexibility in model choice.
  • Document Handling: Features a backend process to ingest PDFs, split them into chunks, embed them using SentenceTransformers, and store the embeddings in a FAISS index for fast retrieval.
  • Interactive Web UI: Built with HTML, CSS, and JavaScript, the frontend provides a user-friendly chat interface, allowing document uploads, model selection (GPT-4/Groq), dark/light mode toggle, and chat history display.
  • Utility Features: Includes real-time display of token usage and API latency, and functionality to export the entire conversation history.

DailyHarvest – Grocery Delivery App

A modern, full-stack grocery delivery application built with React and Supabase. Designed to provide a seamless shopping experience with features for both customers and administrators, demonstrating proficiency in building scalable, user-centric web applications with robust backend services.

Key Features & Highlights

  • Secure Authentication: Implemented user registration and login using Supabase Authentication, ensuring secure access to the application.
  • Dynamic Product Catalog: Allows users to browse a variety of products (vegetables, fruits, meat/seafood) fetched from a Supabase PostgreSQL database, complete with images, descriptions, and pricing.

  • Order Management: Enforces a $25 minimum order requirement and provides a streamlined checkout flow, ensuring business logic is correctly handled.
  • Admin Dashboard: A dedicated interface for administrators to perform CRUD operations on products, manage inventory levels, and view all customer orders, demonstrating role-based access control.
  • Responsive & Accessible Design: Built with a mobile-first approach using Tailwind CSS, ensuring the application is visually appealing and fully functional across all device sizes and user preferences, including a system-aware dark/light mode toggle.

MediTrack – Secure Medical Record Management

A secure medical record management system designed for clinics and hospitals. It provides a centralized digital platform for managing patient data, visit histories, symptoms, and prescriptions, emphasizing data security, integrity, and role-based access control essential for healthcare applications.

System Design & Security

  • Secure Role-Based Access Control (RBAC): Implemented using Spring Security and JWT (JSON Web Tokens). Users authenticate securely, and their roles (Doctor or Administrator) strictly control their access to different parts of the application and API endpoints, protecting sensitive patient health information (PHI).
  • Patient Management: Provides a central repository for patient information, including name, date of birth (with automatic age calculation), email, and phone number. Ensures accurate and easily accessible patient profiles.
  • Comprehensive Medical Records: Allows doctors to document each patient visit in detail, including attending doctor, timestamp, reported symptoms, and prescribed treatments. Maintains a complete, chronological history for each patient, crucial for ongoing care.
  • Reliable Backend API: Built with Java 8 and Spring Boot, providing a robust and maintainable RESTful API. Handles business logic, data processing, and integrates securely with the PostgreSQL database.
  • Relational Database Design: Utilizes a normalized PostgreSQL schema (Users, Patients, Visits tables) to ensure data integrity, reduce redundancy, and enable efficient querying of complex relationships (e.g., retrieving a patient's full history).
  • Modern Frontend: Developed with React and TypeScript for a responsive and type-safe user interface. Uses Tailwind CSS for styling and shadcn/ui components for a polished look, providing an intuitive experience for healthcare professionals.
